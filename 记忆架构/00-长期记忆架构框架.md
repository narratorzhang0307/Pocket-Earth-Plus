# Pocket Earth · 长期记忆架构框架

> 由「深读 hermes-agent + openhanako + 现状盘点 + 黄佳两本书记忆哲学 → 综合 → 对抗校验」得出。
> 约束:纯端侧 PWA + 云脑 DeepSeek、无持久后端 DB、iOS PWA 存储/后台限制、绝不动 FROST 封闭枚举内核、不碰 GitHub。

## 一、直接回答三个问题

**① 目前配了长期记忆吗?** — 配了,但只配了「半条腿」。
唯一真正的跨会话长期记忆是 `frost-agent/harness/profile.ts`(localStorage `pe.profile.v1`):按 `domain(movies/books/music/photos/travel) → field → TagCount[]` 存**脱敏标签计数**(导演/作者/艺人/流派/城市/国别…),每字段 ≤50 条按热度截断;另有一句话口味叙事 `pe.profile.narrative.v1`(`summarizeTaste`,带 fingerprint 缓存)。隐私边界很干净:**只云脑侧读,端侧 Selector 绝不接触**。另有一条不持久的并行口味 `src/app/lib/taste.ts`(仅 books 域、读静态 `bookRecords`)。

**② 是挂在总 FROST 上的吗?** — 是。
`profile.ts` 就挂在 frost-agent harness 上,是名正言顺的总 FROST 长期记忆层,已被 `AgentChat`(各 curator 共用读出口)、`heartbeat`、广场页读取。**你记得的「做长期记忆时 frost-agent 还没做好」如今不成立**——harness 已有 profile/heartbeat/skillForge/persona 等部件。真正问题不是「没挂」,而是挂得**太细、太单向**:
- 只有一根细线:增量写仅覆盖 `movie.country / book.author / music.cities+artists / travel.cities`,profile 设计的 `directors/genres/moods/storyPlaces` 等字段几乎从不被写;
- 单向且漏接:photo geoPin **完全不调** `recordSignals`、council 判例 / bookNotes / 各 ratingFix 纠错信号全不回流;
- 没有「记忆中枢」抽象:`AgentChat` 手工拼 `getProfileSummary + getTasteSummary` 两段,新增 agent 容易漏(photo/council 已经漏了)。
结论:**保留并强化这一层,不要另起炉灶**。

**③ 子 agent 要配长期记忆吗?** — 要,但配的是「任务侧领域记忆」,不是「跨域用户画像」。
用黄佳的**记忆四象限**判:**自我记忆(Self:跨域偏好/风格/去过的地点,属于「人」)归总 FROST;任务知识(Task:本域规则/阈值/情节库,换域就没用)归子 agent**。现状里 photo/movie/book 的端侧 store + 纠错偏好已是「子 agent 领域记忆」的雏形,方向正确;缺的不是「要不要有」,而是「**子 agent 的脱敏稳定结论要经总控回流共享层**」这条数据流——目前几乎断裂。子 agent 之间**绝不互读**对方私有记忆(Harness:报文传输而非共享内存),只把脱敏结论经总 FROST 中转。

## 三、推荐架构:三层 + 一中枢(全端侧)

- **L1 总 FROST 记忆层 = 宽而稳的「自我记忆」**:属于「人」的跨域画像。**复用并扩展现有 `profile.ts`**(不另起):(a) 标签计数维持现状(机器可聚合、便宜);(b) **扩写现有 `summarizeTaste`**(不是新建!)从一句话升级为 today/longterm/facts 三段自然语言用户模型,会话后由 DeepSeek 压缩、内容指纹没变就 skip。写权限只归总 FROST(经回流中枢)。读出口唯一,经记忆中枢拼装注入云脑 system。
- **L2 子 agent 领域记忆层 = 深而窄的「任务知识 + 情节库」**:保留各子 agent 现有端侧 store,标准化为「领域索引 + 领域规则/阈值 + 领域笔记/判例」。增强(P1):给 council/photo 补**反思记忆**(用户每次纠正→自然语言 Hint,成功固化 Skill、失败固化 Rule,用 SkillForge 已有白名单+reviewSkill 安全闸),但**只存本域**。隔离:原图/CLIP/精确轨迹永留端侧、绝不进 L1;子 agent 间不互读。
- **L3 共享世界知识层 = 与个体无关的语义知识**:地名/坐标/影史,多 agent 共享只读(现状 `geocodeCity`/`userMarks` 已是雏形),不掺用户偏好。
- **中枢 `memoryRouter`(新增,唯一新模块)**:
  - **读装配** `assembleMemory(domain)`:取代 `AgentChat` 手工拼接,按 scope 拼【L1 标签摘要 + L1 narrative + 本域 L2 摘要 + L3 世界知识】+「记忆即空气」规则,一行注入。
  - **写回流** `contributeToProfile(domain, signals)`:子 agent → L1 的**唯一受白名单约束**写口(物理边界:子 agent 只能写脱敏标签,不能改 User 级 narrative)。

**数据流(单向收敛、报文中转)**:子 agent 运行(L2 私有,高噪声)→ 会话结束复盘提炼脱敏稳定结论 → 经 `memoryRouter.contribute`(白名单)写 L1 → 下次任何 agent 经 `assemble` 读到 → 「一次交互,全域认知」(photo 钉过的城市,travel/电台直接复用),L2 私有库彼此隔离不互污染。

## 四、可直接搬 / 改造搬清单

**Hermes(Nous Research,重型 CLI agent)**
- `direct-copy` **冻结快照式长期记忆**:会话启动读画像拼进 system 并冻结,session 内写只落 localStorage、绝不改本轮已发出的 system 串 → 保 DeepSeek prefix cache 省钱省延迟。`profile.ts` 已天然如此。
- `direct-copy` **用户画像 vs 领域笔记两个正交 store**(USER.md vs MEMORY.md):L1 拆「标签计数」+「画像叙事」两份独立预算,写入强制指定 target。
- `direct-copy` **三段式 review 的「该捕获 vs 绝不捕获」语义**(中文照搬):捕获=对风格/口味/流程的稳定偏好与纠正;绝不捕获=一次性任务叙事、瞬时错误、**对功能的负面断言**(会硬化成几个月后反咬自己的自缚约束)、易重新发现的事实;「Nothing to save 是合法但非默认」。
- `direct-copy` **生命周期纯时间函数** active→stale→archived,只归档不删除、pinned 豁免(零 token)。
- `adapt` **周期 nudge → 旁路「复盘」**把学习与干活解耦:端侧无子进程,改成会话/策展结束回调里发一次无 UI 的轻量 DeepSeek fetch。
- `adapt` **FTS5 跨会话检索的「四模式 + bookend 窗口」**:端侧用 IndexedDB + minisearch,**CJK 必须上 trigram/2-gram 分词**否则中文搜不准。(P1)

**OpenHanako(Electron 桌面 agent)**
- `direct-copy` **「记忆=用户模型,严禁记执行细节」硬边界 prompt**:回答「用户是谁」才抽、回答「怎么干活」不抽、拿不准一律不抽、宁漏不错。
- `direct-copy` **「记忆即空气」注入规则**:禁止暴露记忆痕迹(不说「我记得/你之前说过」)、信息冲突以当前对话为准、不用旧记忆纠正用户。
- `adapt` **分层 markdown today/longterm/facts + 指纹 skip**:这是 Hanako「近期记得牢」的真正来源;端侧用简单 hash 当指纹省调用。
- `adapt` **标签化事实 + 标签检索、弃用向量**:L2 笔记/判例存 `{fact, tags[], time}`,标签交集匹配 + 子串兜底,纯 JS 倒排,零向量库。
- `adapt` **Agent 即文件夹**(身份/能力/记忆按序拼成 system + 可导出):天然支持「导出我的电影口味档案」。(P2)

**Honcho(Hermes 可选支柱)**:依赖常驻后端服务 + 自己的 DB,纯端侧 PWA **照搬必死**,框架已规避,用「会话结束旁路复盘 fire-and-forget」替代。
