# Pocket Earth · 旅行 Agent（travel-curator）架构总纲

> 经多 agent 调研（GitHub 现成项目 + 旅行 MCP 生态 + 仓库现状盘点 + 截图提炼方案）→ 综合草案 → 逐行核对代码的对抗评审 → 终版。7 个 agent / 42 万 token。
>
> **一句话**：不嵌任何外部 repo（都嵌不进 React+TS PWA，只借结构思想）；B 线（规划）其实已上线、本次只做增量；A 线（截图存档）零依赖、P0 先手动录入版；MCP 全部 P1、P0 零 key 也能跑通可见闭环。守隐私铁律、agent 绝不代下单、FROST 封闭枚举零改动。

## 0. 三个反预期结论（先看这个）

1. **「找个现成的改改」基本落空**。GitHub 高 star 的旅行 agent（top3：`google/adk-samples` 9.6k★、`Haohao-end/Ctrip-Style-AI-Travel-Assistant` 55★、`HarimxChoi/langgraph-travel-agent` 15★）**全是 Python / Vertex / 后端**，嵌不进我们的 React+TS PWA；强行移植成本 > 我们自建。**结论：一行不抄（也顺带规避 license），只借它们的结构思想。**

2. **B 线（待出行规划）其实已经上线了**。`src/app/components/TravelRunPage.tsx`（190 行）已跑通整条：端侧排 POI → 分天 → `addUserMark(kind:'travel')` 钉地球 → `recordSignals('travel')` 回流。所以这条线**不是从零写，是增量改造**（抽 lib 六层 + 接跨域画像 + 修一个假排序 bug + 扩城）。

3. **截图脱敏比想象的难，需修正之前「二维码天然安全」的乐观说法**。二维码是图像区域、正则只管已 OCR 出的文本；若留票据缩略图当封面，二维码缩到 160px 仍可扫、身份证号区仍在图上。**正解：票据原图根本不持久化，只存脱敏后的结构化字段，封面改用城市占位图。** 且 OCR 一旦错，DeepSeek 不收图 + 隐私禁传图 = 没有云脑兜底，唯一兜底是用户手改。

## 2. 旅行 MCP 生态结论

- **Anthropic 官方 MCP Registry 的旅行域是空的**（只收 Asana/Jira/Slack 类企业连接器）。所以旅行 MCP 全是 GitHub 社区 / 各厂自建，**没有「官方一键接」的现成连接器**。
- **真·零 key 能经 `server.mjs` 代理跑、纯查询的只有 4 类**：
  - **OSM**（地图/路线/POI，无 key、合规）— P0/P1 主力
  - **Open-Meteo**（天气，无 key，但仅非商用）
  - **Airbnb**（openbnb，无 key，但是**抓取**，合规/稳定性自评）— 仅参考+跳链接
  - **fli**（航班，无 key，但**逆向 Google Flights**，随时挂）— 仅「参考价」+跳链接
- **要 key 但免费注册级**：Foursquare（POI/餐厅，官方）、Amadeus（航班+酒店报价，test 免费/生产付费）、Kiwi（官方，返回结果自带「官网下单链接」，设计最贴红线）。
- **火车/地面交通是最大空白**：没有全球统一源，全是按国家的低成熟度社区项目；先用 OSM/OSRM 路线近似。
- **红线落地**：所有真实下单/支付端点（Amadeus book、Duffel 下单、任何 OTA 创建订单）**一律不暴露给 agent**；航班→Kiwi/fli 出链接、酒店→Airbnb/Amadeus 出清单+链接，**支付永远由用户自己在对方站点完成**。
- **P0 不接任何 MCP**：纯本地 catalog + 云脑/端侧排序，零 key 先跑通闭环；MCP 全在 P1，经 `server.mjs` 代理（前端直连会泄 key/撞 CORS）。

## 4. 两条线流水线

### B 线（规划）— P0 主路径（已上线，增量版）
```
输入: 去京都 / 4天 / 偏好[美食,小众]
 └ sense: 解析 {dest,days,prefs}; geocodeCity 反查坐标(确定性,不上网)
 └ memoryRouter.assembleMemory(): 取电影/读书/音乐+travel 跨域画像  ← 本次新增
 └ plan(吃画像打分): pickDestination → pickPOIs → pickFood → planDays
     ⚠ 排序后端三级: 本地端侧真后端 → 云脑 /api/frost-llm 按画像排 → 本地命中度
        （线上 /api/edge 默认 stub=假分,不能靠它,见风险表）
 └ (P1) tagging.enrichTrip 云脑补非主流景点; mcp 查交通/酒店"参考价"+链接
 └ 产出逐日方案 →「完成行程·钉星球」→ pin 多点钉地球 → recordSignals 回流
 ※ 绝不代下单,只给官网链接
```

### A 线（存档）— P1（P0 仅手动录入版，无 OCR）
```
P0(手动): 手填 城市/日期/交通/住宿 → pin 钉点
P1(OCR):
 截图(车票/酒店/门票混杂)
  └ (可选)CLIP 筛单据 [模型从 CDN 拉,iOS 离线/被墙会失败→跳过走手填兜底]
  └ tesseract.js chi_sim OCR(接受低精度) → 纯文本
      └ 端侧正则脱敏: 身份证(18)/手机(11)打码; 强字段(车次/航班/日期/金额)正则锁 confidence=1
  └ tagging: 脱敏文本 → /api/frost-llm 归类补软字段(代码层白名单 merge 防覆盖)
  └ cluster: 并查集聚类 → TripArchive[]   [OCR 错无云脑兜底,只能用户手改]
  └ critic: 去重+跨段一致性 → pin: 低置信标黄 → 确认 → 多点钉地球
```

## 5. 截图提炼 + 脱敏 + TripArchive schema（P1）

**唯一合规主路径 = 端侧 OCR → 端侧脱敏+正则锁字段 → 脱敏文本喂 `/api/frost-llm`。**

- **OCR**:P1 引入 `tesseract.js` chi_sim 当真依赖（接受 ~20% 低精度，强字段标黄人工核）；PP-OCRv5 WASM 成熟度未证实，列 P1+ 自验后替换。
- **脱敏（关键修正）**:正则只管已 OCR 文本；二维码无版面检测无法定位。→ **票据原图不持久化、cover 用城市占位图、整张图丢弃只留脱敏字段。**
- **防幻觉**:车次/航班/日期/金额走端侧正则锁定，**防覆盖在代码合并层做（白名单只允许 LLM 写软字段），不写进 prompt 当保证。**
- **无二次纠错通道**:DeepSeek 不收图 + 隐私禁传图 → OCR 错误无模型兜底，唯一兜底=用户手改。

```ts
TripArchive { id, title, dateStart, dateEnd, cities[], cover?(城市占位图,非票据原图), createdAt }
segments[]: { id, mode:'train'|'flight'|'bus'|'car', code, date, depTime?, arrTime?,
              from:{name,city,lat?,lng?}, to:{...}, confidence:0-1 }   // 不存 raw 票据图
stays[]:    { id, hotel, city, checkIn, checkOut, geo?, confidence }
spots[]:    { id, name, city, date?, geo?, confidence }
```
geo 复用 `geoStickers.geocodeCity`（确定性、不上网）；解析不到标 `needGeo` 让用户 UI 补（等同 photo 的 `needPlace`）。

## 6. 钉地球对接（多点散钉现成 / 连线是新工作量）

- **多点散钉（P0，现成）**:每个 stop/stay 钉点，`addUserMark(kind:'travel')` + `spreadCoord` 散开。`MarkerKind` 已含 `'travel'`，`MyMapTab` 已订阅 userMarks 自动渲染——agent 只当生产者。
- **连线（P1，新工作量）**:`MyMapTab.tsx:46-54` 把 userMarks 一律转成 Point，**没有 LineString 图层**；那条可见折线是**写死的西湖 SVG 标注**，与 tripId 无关。→ 新增 `lines.ts` + 在 `toFeatureCollection` 旁加 `tripLines` LineString source + `addLayer({type:'line'})`，按 `meta.tripId` 聚合连点；`status:'future'` 用虚线区分「已去 vs 计划中」。
- **整程卡（P1，新工作量）**:`MarkerDetail.tsx:148` 的 travel 卡是**单点卡**，无 tripId 聚合；P0 复用单点卡，P1 扩整程卡。

```ts
// travel meta（UserMark.meta 自由字段，无需改类型）
meta: { tripId, role:'seg-from'|'seg-to'|'stay'|'spot', city, date,
        mode?, code?(脱敏), hotel?, confidence, status:'done'|'future' }
```
分享地球时坐标走 `coarsenForShare` 抽稀防轨迹泄露。

## 7. 记忆/画像注入（assembleMemory + profile travel 字段 + 回流）

- **注入（读）**:B 线 `plan.ts` 规划前调 `memoryRouter.assembleMemory()`（签名不动），把电影/读书/音乐+travel 跨域画像拼进排序/云脑 prompt。**本次新增**——现在 `TravelRunPage` 只用本地 prefs，没注入跨域画像。
- **回流（写）**:`pin.ts` 落地后 `recordSignals('travel', fields)`，只在 `FIELD_LABEL` 补中文标签、**不扩 domain union**：
  - `cities` 常去城市 / `prefs` 旅行偏好 / `modes` 交通偏好 / `seasons` 偏好季节 / `moods` 旅行心情
- **回流原则**:只回流已确认钉地球的确定性结论；不回流单次低置信 OCR 字段、未确认的规划建议。critic 5★ 按 movie 模式加权。
